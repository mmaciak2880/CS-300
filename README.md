The scenario for these two projects presented a problem needing to be solved for ABC University. ABCU wanted a program that would help their computer science department search for course information and see all courses in order with their prerequisites. In order to tackle this problem I needed to decide on a data structure to use to hold all of the courses. Because the data would need to be sorted in alphanumeric order at some point. I decided to use a binary search tree due to the fact that everything would be kept sorted as courses are added to it. The main challenge that I faced when implementing this program was creating the section that would check the input data format, check that any prerequisites are already in the BST, add the course if ok, and not if formatting errors are present. While creating this section I ended up over complicating my solution which also had a few errors. Due to me making it over complicated it was hard to see what was causing these errors, so my solution was to start over with more simple/organized code. Testing more often than with my previous attempt to implement this section. Leading to the successful implementation of this section and the program as a whole. My work on this project made me think about how to implement different data structures and which ones are the best to use depending on the problem at hand. Which has led me to think about how to store data when designing a program and not just the fact that data needs to be stored.
